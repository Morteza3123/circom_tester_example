# Multiplier2 Circuit Test Report

## Overview

This report documents the issues encountered and resolved while setting up and running the `Multiplier2` circom circuit tests using `mocha` and `circom_tester`.

---

## Test Environment

- **Circuit:** `Multiplier2.circom` — a simple zk-SNARK circuit with two inputs (`a`, `b`) and one output (`c = a * b`)
- **Test file:** `multiplier2.js`
- **Test framework:** Mocha + Chai
- **Tester library:** `circom_tester` (WASM and C++ backends)

---

## Problems Encountered & Solutions

### Problem 1: `mocha` not found globally

**Error:**
```
bash: mocha: command not found
```

**Cause:** Mocha is installed locally (not globally), so the `mocha` binary is not on PATH.

**Fix:** Use `npx` to invoke the local installation:
```bash
npx mocha -p 'multiplier2.js'
```

---

### Problem 2: `ts-node` not installed

**Error:**
```
ERROR: Error [ERR_MODULE_NOT_FOUND]: Cannot find package 'ts-node'
```

**Cause:** The original command included `-r ts-node/register`, which is only needed for TypeScript test files. `multiplier2.js` is plain JavaScript.

**Fix:** Drop the `-r ts-node/register` flag entirely — it is not needed for `.js` files.

---

### Problem 3: Wrong `require` path for `circom_tester`

**Error:**
```
Cannot find module './../../index'
```

**Cause:** The test file used a relative path `require("./../../index")`, which was written to run from inside the `circom_tester` package's own test directory. When run as a standalone project, that path doesn't resolve.

**Fix:** Updated `multiplier2.js` to use the installed package name instead:
```js
// Before
const wasm_tester = require("./../../index").wasm;
const c_tester = require("./../../index").c;

// After
const wasm_tester = require("circom_tester").wasm;
const c_tester = require("circom_tester").c;
```

---

### Problem 4: Missing C++ dependency — `nlohmann/json`

**Error:**
```
fatal error: nlohmann/json.hpp: No such file or directory
```

**Cause:** The C++ witness calculator generated by circom requires the `nlohmann/json` header library, which was not installed on the system.

**Fix:**
```bash
sudo apt-get install -y nlohmann-json3-dev
```

---

### Problem 5: Missing C++ dependency — `gmp.h`

**Error:**
```
fatal error: gmp.h: No such file or directory
```

**Cause:** The circom C++ runtime (`circom.hpp`) depends on the GNU Multiple Precision Arithmetic Library (GMP).

**Fix:**
```bash
sudo apt-get install -y libgmp-dev
```

---

### Problem 6: Missing assembler — `nasm`

**Error:**
```
make: nasm: No such file or directory
make: *** [Makefile:19: fr_asm.o] Error 127
```

**Cause:** The circom C++ build uses NASM (Netwide Assembler) to compile field arithmetic assembly (`fr_asm.s`). It was not installed.

**Fix:**
```bash
sudo apt-get install -y nasm
```

---

### Problem 7: Stale `tmp/` build artifacts

**Error:**
```
/bin/sh: 1: .../tmp/Multiplier2_cpp/Multiplier2: not found
```

**Cause:** A previous failed C++ compilation left a partial `tmp/` directory. The "recompile: false" test then tried to run a binary that was never successfully built.

**Fix:** Delete the `tmp/` directory before re-running tests to force a clean rebuild:
```bash
rm -rf tmp/
```

---

## Final Working Command

```bash
rm -rf tmp/ && npx mocha -p 'multiplier2.js'
```
